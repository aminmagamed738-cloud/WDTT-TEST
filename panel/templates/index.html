Гайд: 3x-ui + WDTT Panel Как направить клиентский трафик через VK-звонки
Схема работы:
Клиент → 3x-ui (VLESS/VMess) → SOCKS5-прокси WDTT → WireGuard-туннель → VK TURN-сервер → WDTT-сервер → Интернет

Шаг 1 — Создайте подключение в панели

  1. Вставьте ссылки VK-звонков (vk.com/call/join/...)
  2. Укажите адрес вашего WDTT-сервера IP:56000
  3. Введите пароль подключения с WDTT-сервера
  4. Нажмите «Запустить» — дождитесь статуса ● подключено
  5. Запишите SOCKS5 порт (он слушает только на 127.0.0.1)

Шаг 2 — Настройте 3x-ui (Outbound)

В панели 3x-ui перейдите: Настройки → Xray → Outbounds, добавьте новый:

{
  "tag": "wdtt-out",
  "protocol": "socks",
  "settings": {
    "servers": [
      {
        "address": "127.0.0.1",
        "port": SOCKS_PORT,
        "users": []
      }
    ]
  }
}

Шаг 3 — Настройте Routing в 3x-ui

Перейдите: Настройки → Xray → Routing Rules.

Важно: правило ниже направляет через WDTT только российские адреса (geoip:ru / geosite:ru). Чтобы проверить туннель командой curl или перенаправить весь TCP-трафик, используйте второй вариант — без фильтра по IP/домену.

Вариант A — только RU-трафик:

{
  "type": "field",
  "outboundTag": "wdtt-out",
  "ip": ["geoip:ru"],
  "domain": ["geosite:ru"]
}

Вариант B — весь TCP (для теста или полного проксирования):

{
  "type": "field",
  "outboundTag": "wdtt-out",
  "network": "tcp"
}

SOCKS5 доступен только локальным сервисам этого VPS. Не открывайте его наружу через firewall или reverse-proxy без отдельной аутентификации.

Шаг 4 — Проверка SOCKS5 напрямую

Эта команда обращается к SOCKS5 напрямую с VPS — минуя 3x-ui. Она работает независимо от routing rules в Xray и проверяет именно WireGuard-туннель:

curl --socks5-hostname 127.0.0.1:SOCKS_PORT https://ifconfig.me

Должен вернуться IP WDTT-сервера (не IP вашего VPS). Если соединение отклоняется — туннель не поднят; если возвращается IP вашего VPS — WireGuard работает, но профильный маршрут не применился (проверьте ip rule show на сервере).

Дополнительно — несколько VK-звонков

Добавьте несколько ссылок в одном профиле — они будут работать параллельно, увеличивая пропускную способность (агрегация каналов через разные TURN-серверы VK). Рекомендуется 3–5 звонков для стабильной работы.
